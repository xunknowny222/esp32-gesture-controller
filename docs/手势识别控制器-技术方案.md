## 一、项目概述

### 1.1 产品定位

一款**便携式、低功耗、离线运行**的手势识别控制器，通过内置摄像头采集手势图像，在 ESP32-S3 本地完成 AI 推理，识别预设手势并输出控制指令，可用于：

- 智能家居控制（灯光、空调、窗帘）
- 无人机/机器人手势遥控
- 演示/展览交互设备
- 无障碍辅助控制

### 1.2 核心创新点

| 创新点 | 说明 |
|--------|------|
| **边缘 AI 推理** | 所有识别在本地完成，无需云端，延迟 < 200ms |
| **超低功耗** | 深度睡眠 + 手势唤醒，待机电流 < 50μA |
| **便携式设计** | 电池供电，体积 < 100×60×20mm，可手持/佩戴 |
| **多协议输出** | Wi-Fi/MQTT、BLE、红外、UART 多种控制方式可选 |

## 四、软件方案

### 4.1 软件架构

```
┌─────────────────────────────────────────────────────┐
│                  应用层 (Application)                │
│   手势命令解析 → 控制指令输出 → 设备联动逻辑        │

                         │
┌─────────────────────────────────────────────────────┐

                         │

```

### 4.2 手势识别模型方案

| 方案 | 模型 | 精度 | 推理时间 | 难度 |
|------|------|------|----------|------|
| **方案 A（推荐）** | MobileNetV1 量化版 | ~92% | ~150ms | ⭐⭐ |

**推荐方案 A**：乐鑫 ESP-DL 库已提供预训练模型，可直接部署。

## 五、关键功能实现

### 5.1 图像采集与预处理

```c
// 摄像头配置（ESP-IDF）
camera_config_t config = {
    .pin_pwdn = -1,
    .pin_reset = -1,
    .pin_xclk = CAM_XCLK_PIN,
    .pin_sscb_sda = CAM_SIOD_PIN,
    .pin_sscb_scl = CAM_SIOC_PIN,
    .pin_d7 = CAM_D7_PIN,
    // ... 其他引脚配置
    .pixel_format = PIXFORMAT_GRAYSCALE,  // 灰度图加速推理
    .frame_size = FRAMESIZE_QVGA,         // 320x240
    .jpeg_quality = 10,
    .fb_count = 2
};
```

### 5.2 模型部署流程

1. **训练阶段**（PC 端）
   - 使用 TensorFlow/PyTorch 训练手势识别模型
   - 导出为 TFLite 格式
   - 使用 ESP-DL 工具量化（INT8）

2. **部署阶段**（ESP32-S3）
   - 将模型烧录到 Flash
   - 使用 ESP-DL API 加载模型
   - 推理结果映射为控制命令

### 5.3 低功耗设计

| 模式 | 功耗 | 触发条件 |
|------|------|----------|
| 深度睡眠 | ~50μA | 待机状态 |
| 唤醒检测 | ~5mA | PIR 传感器/触摸唤醒 |
| 推理运行 | ~120mA | 摄像头 + AI 推理 |
| 无线传输 | ~200mA | Wi-Fi 发送峰值 |

**电源策略**：
- 默认深度睡眠，触摸/PIR 唤醒
- 单次识别完成后自动休眠（5 秒无操作）
- 电量低于 20% 时禁用 Wi-Fi，仅保留 BLE

## 七、风险评估与应对

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| 模型精度不足 | 中 | 高 | 增加训练数据、数据增强、调整模型结构 |
| 推理速度过慢 | 中 | 中 | 降低输入分辨率、模型量化、使用向量指令集 |
| 功耗过高 | 低 | 中 | 优化睡眠策略、降低摄像头帧率 |
| 硬件成本超预算 | 低 | 低 | 批量采购、简化外壳设计 |
| 开发周期延误 | 中 | 高 | 每周检查进度、优先完成核心功能 |
