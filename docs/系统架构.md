```
                     ┌──────────┐  ┌──────────┐

                     │云平台    │  │电视/空调 │
                     └──────────┘  └──────────┘
```

### 架构说明

1. **输入层**：用户通过手势输入控制意图
2. **感知层**：摄像头采集手势图像
3. **处理层**：ESP32-S3 运行手势识别模型
4. **反馈层**：OLED 显示识别结果和系统状态
5. **执行层**：通过 Wi-Fi/MQTT 或红外控制设备

## 4. 手势识别方案

### 4.1 支持的手势定义

| 手势 ID | 手势名称 | 控制功能 | 优先级 |
|--------|---------|---------|--------|
| 0 | 👊 握拳 | 关闭所有设备 | P0 |
| 1 | ✋ 张开 | 打开所有设备 | P0 |
| 2 | ⬅️ 左滑 | 亮度-/音量- | P1 |
| 3 | ➡️ 右滑 | 亮度+/音量+ | P1 |
| 4 | ⬆️ 上滑 | 上一个设备 | P2 |
| 5 | ⬇️ 下滑 | 下一个设备 | P2 |

---

### 4.2 技术方案对比

| 方案 | 准确率 | 推理速度 | 内存占用 | 开发难度 | 选择 |
|------|--------|---------|---------|---------|------|
| 传统图像处理 | 70-80% | <50ms | <100KB | 低 | ⬜ |
| 深度学习 (从头训练) | 90-95% | <200ms | ~500KB | 高 | ⬜ |
| 预训练模型 + 微调 | 85-90% | <300ms | ~300KB | 中 | ✅ |

**选择理由**：
- 时间紧张（4 周冲刺）
- 预训练模型开发快
- 准确率满足比赛要求

## 5. 电源管理

### 5.1 功耗分析

| 工作模式 | 电流消耗 | 说明 |
|---------|---------|------|
| 深度睡眠 | ~10μA | 仅 RTC 运行 |
| 待机模式 | ~10mA | Wi-Fi 监听 |
| 正常运行 | ~100mA | 摄像头 + Wi-Fi |
| 推理峰值 | ~150mA | 模型推理中 |

---

### 5.2 续航估算

**电池参数**：
- 容量：500mAh
- 电压：3.7V
- 能量：1.85Wh

**使用场景**：
- 日常使用（间歇工作）：平均 50mA → **续航约 10 小时**
- 连续使用：平均 100mA → **续航约 5 小时**
- 深度睡眠：~10μA → **续航约 5000 小时**

---

### 5.3 低功耗策略

```cpp
// 伪代码示例
void enter_sleep_mode() {
  // 1. 关闭摄像头
  esp_camera_deinit();
  
  // 2. 关闭 Wi-Fi
  WiFi.disconnect();
  
  // 3. 关闭 OLED
  display.ssd1306_command(SSD1306_DISPLAYOFF);
  
  // 4. 进入深度睡眠
  esp_deep_sleep(10000);  // 10 秒后唤醒
}
```

## 8. 风险评估与应对

### 8.1 风险矩阵

| 风险 | 概率 | 影响 | 风险等级 | 应对措施 |
|------|------|------|---------|---------|
| 物流延误 | 高 | 高 | 🔴 高 | 先用开发板，外设后加 |
| 模型精度不足 | 中 | 高 | 🟡 中 | 减少手势种类（6→3） |

| Wi-Fi 不稳定 | 中 | 中 | 🟡 中 | 准备离线演示模式 |
